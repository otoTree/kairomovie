# 画布节点可编辑与元信息入上下文设计

## 目标

- 让画布里的文本节点可直接编辑内容
- 让用户在节点悬浮时补充节点元信息
- 让被标记的节点元信息进入 Agent 每次输入上下文
- 保持现有画布快照存储结构兼容

## 现状问题

- 节点容器整体绑定了拖拽起手，导致文本输入焦点经常被拦截
- 节点缺少“是否参与 Agent 上下文”的显式开关
- 节点附加描述未被系统化地注入到 `/api/v1/agent/chat` 的 prompt

## 方案总览

### 1) 节点编辑交互

- 将拖拽触发区域收敛到节点标题栏
- 对 `textarea/input/button/video` 等交互元素跳过拖拽逻辑
- 文本节点继续使用内嵌 `Textarea`，直接更新 `nodes` 状态

### 2) 节点元信息模型

在 `CanvasNode` 增加两项可选字段：

- `contextEnabled?: boolean`：是否注入 Agent 上下文
- `contextLabel?: string`：该节点在上下文中的标题

同时复用已有字段：

- `content?: string`：作为“核心元信息”正文

这样不引入额外表结构，仍由 `snapshot.nodes` 持久化。

### 3) 悬浮编辑 UI

节点悬浮时在节点内部展示“上下文配置区”：

- 注入开关（Switch）
- 元信息标题（Input，可选）
- 核心元信息正文（Textarea）

配置区写回当前节点，立即反映在内存状态与后续保存快照中。

### 4) Agent 上下文注入

前端发送聊天请求前构建 `canvasContext`：

- 仅收集 `contextEnabled === true` 的节点
- 节点内容按结构化文本拼接：节点类型、标题、核心元信息、节点主内容
- 限制节点数量上限，避免 prompt 膨胀

后端 `/api/v1/agent/chat` 增加 `canvasContext` 入参，并拼接为增强 prompt：

- 有上下文时：`【画布节点上下文】...【用户输入】...`
- 无上下文时：保持原始 prompt

## 数据结构兼容性

- 新字段为可选字段，历史快照可直接读取
- `toCanvasNode` 在反序列化时做类型保护，缺失字段自动回退
- 旧版本节点不影响渲染与聊天

## 行为边界

- 默认不注入任何节点，避免误导 Agent
- 用户显式打开开关后才进入上下文
- 元信息为空时仍可注入节点主内容，保证最低可用

## 后续可演进项

- 增加“节点上下文优先级”用于排序注入
- 支持按会话缓存最近一次上下文摘要
- 在聊天面板展示“本次注入上下文预览”
